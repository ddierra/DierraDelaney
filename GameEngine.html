Project Overview

Gamer Deedz Engine is a 2D game engine built using C++ and the SFML library as its core. As of today, the engine supports basic window creation, input handling, and shape rendering with keyboard-controlled movement.

The project is a work in progress, with plans to implement additional features such as sprite rendering, animations, sound integration, and a scene management system.

Technologies Used

  • C++
  • SFML (Simple and Fast Multimedia Library)
  • CMake
  • Git for version control

Basics (Weeks 1-4)

This section covered a period closer to a few hours than four weeks, but as the timeline specifies the first four weeks of development, I’ll keep it this way. During this time, I focused on establishing the foundational features of the engine. This included setting up the project structure, integrating the SFML library, and implementing basic functionalities such as window creation, input handling, and shape rendering.

By the end of this section, users could create a window, render simple shapes (a blue oval and a yellow square), and control their movement using keyboard inputs.

Entity-Component-System (ECS) Skeleton (Week 5)

In this phase, I began laying the groundwork for an Entity-Component-System (ECS) architecture. This involved designing the basic structure for entities, components, and systems, which will allow for more complex game objects and behaviors to be created with ease.

ECS is a powerful design pattern that promotes flexibility and scalability in game development. By separating data (components) from behavior (systems), it allows efficient management of all game entities. At this stage, the engine only has a basic, functional ECS skeleton and doesn’t yet perform full gameplay logic.

Here’s a simplified snippet illustrating the structure of the ECS components implemented during this phase: 


          // Simple ECS header snippet

          #include 
          #include 

          using Entity = unsigned int;

          class EntityManager {
            public:
              Entity createEntity();
              void destroyEntity(Entity e);

              template
                void addComponent(Entity e, Args&&... args) {
                  components[typeid(Component)][e] = Component(std::forward(args)...);
                  }

            private:
              std::unordered_map> components;
            };

In this system:

  • Entities are unique identifiers for game objects.
  • Components store data (like position, velocity, or health) associated with entities.
  • Systems will later define how components interact and update behavior.

This ECS skeleton sets the foundation for future features, such as rendering systems, movement systems, and sprite management, which will be added in the upcoming weeks.

Next Steps

Moving forward, I will expand the Entity-Component-System (ECS) skeleton into fully functional systems, starting with a render system to draw entities on screen and a movement system to update their positions. This will enhance the engine's flexibility and scalability, enabling the creation of more complex game objects and behaviors, while keeping management of all entities efficient.
